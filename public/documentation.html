Movie API Documentation

Welcome to the Movie API! This API allows developers to access information about movies, including details such as title, genre, release year, and ratings. You can use this API to fetch movie data and integrate it into your applications.

As we continue building this API, more endpoints and features will be added. Check back here for updates and usage instructions.

This API provides endpoints to manage movies and users.

Request URL Query Parameters Request Body Response
GET /movies None None Array of all movie objects, e.g. [{ "title": "Inception", "genre": {...}, "director": {...} }, ...]
GET /movies/:title title - the movie title None Movie object with description, genre, director, image URL, featured flag
GET /genres/:name name - genre name None Genre object, e.g. { "name": "Sci-Fi", "description": "Science fiction movies" }
GET /directors/:name name - director's name None Director object, e.g. { "name": "Christopher Nolan", "bio": "...", "birthYear": 1970, "deathYear": null }
POST /users None { "name": "John Doe", "favoriteMovies": [] } Created user object with unique ID, e.g. { "id": "uuid", "name": "John Doe", "favoriteMovies": [] }
PUT /users/:username username - existing user's username { "name": "New Name" } Updated user object with new name
POST /users/:username/movies/:movieID username - user identifier, movieID - movie title or ID None (data comes from URL) Confirmation text, e.g. "Inception has been added to user JohnDoe's array."
DELETE /users/:username/movies/:movieID username - user identifier, movieID - movie title or ID None Confirmation text, e.g. "Inception has been removed from user JohnDoe's array."
DELETE /users/:username username - user identifier None Confirmation text, e.g. "User JohnDoe has been removed."
